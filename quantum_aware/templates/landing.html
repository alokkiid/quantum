How It Works

Three Autonomous Defense States

The system watches every request. When it sees a threat, it escalates — automatically, without human intervention.

S1 — NORMAL

AES-128-GCM

Files at rest. ML engine silently monitors access patterns, request rates, and auth behaviour in the background.

TRIGGER → Upload · Baseline state
S2 — SUSPICIOUS

AES-256 Escalation

Anomaly detected. Old key destroyed in memory. Re-encrypted with AES-256. Hysteresis timer starts — 5 min clean window needed to revert.

TRIGGER → 5+ auth failures / 60s · ip_rps ≥ 5
S3 — ACTIVE ATTACK

Continuous Rotation

Evasion loop engaged. Decrypt → new key → re-encrypt → overwrite disk → destroy old key → repeat every 4 seconds. Attacker's captured ciphertext is already stale.

TRIGGER → 100+ req/s global · 50+ req/s single IP
Features

Everything You Need for Secure Storage

Built on production-grade cryptography with ML-driven autonomous response.

Zero Plaintext on Disk

All encryption and decryption happens in Flask memory. Plaintext bytes never touch the filesystem at any stage.

Moving Target Defence

Keys rotate every 4 seconds under attack. Any ciphertext an attacker captures is immediately obsolete.

Isolation Forest ML

7-feature anomaly detection trained on real access patterns. Classifies Normal, Suspicious, and Attack states.

Per-User Isolation

Each user's files are independently encrypted with unique keys. No cross-user access, even for admins.

Full Audit Trail

Every KEY_GENERATED, KEY_DESTROYED, STATE_CHANGE, and ROTATION event logged with timestamp. CSV export available.

SHA-256 Fingerprinting

Every uploaded file gets a SHA-256 fingerprint before encryption, providing tamper detection on download.

Architecture

Built for Real Threats

Every layer of the system is designed to assume the attacker has already breached the perimeter.

🧠

ML Threat Engine

7-feature classifier — global rps, per-IP rps, auth failure rate, geo anomaly, time-of-day, UA entropy

🔑

Crypto Engine

AES-GCM with 12-byte nonces. Full key lifecycle — generate, active, destroy. No key reuse ever.

⚙️

State Machine

3-state FSM with hysteresis. Never reverts S3→S1 directly. 5-minute clean window enforced.

🔄

Rotation Loop

Daemon thread per file. Configurable interval (default 4s). Self-terminates when ML clears Attack.

📡

Real-Time Monitor

Background thread evaluates every file every 10s. Before-request hook counts all traffic in-memory.

📋

Audit Trail

Immutable event log. Every key event, state change, and rotation timestamped and exportable as CSV.
